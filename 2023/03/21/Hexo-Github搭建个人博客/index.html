Hexo+Github搭建个人博客

Author: Say Yi Date: March 21, 2023  14:19:45

安装node.js

node.js官网：https://nodejs.org/en

安装git

  • git下载网站：https://git-scm.com/downloads

  • 安装过程中保持默认设置即可，安装完成后查看菜单搜索“git”，查找到即安装成功。

  • 在git登录自己的账号，桌面右键后点击“git bash here”，会跳出命令框，输入下列代码

    git config –global user.name “Your Name”
    gif config –global user.email email@example.com

其中Your Name和 email@example.com 替换成自己注册时的账户名和邮箱

  • 安装git后可通过下列代码检查是否安装成功
    git version
    node -v
    npm -v

安装Hexo

桌面右键“git bash here”，输入下列命令
npm install -g hexo-cli
完成Hexo的安装后，建议在D槽新建一个专属为个人博客的文件夹，我自己的文件夹取名为“perBlog”。
在“perBlog”文件夹下右键点击“git bash here”，执行下列命令，Hexo会在该文件夹下新建所需要的文件
hexo init myBlog
cd myBlog
npm instal

上述命令都没有报错即为成功。目前你的git bash是在/perBlog/myBlog/目录下，运行命令行hexo s ，其中s是server的缩写，即开启服务器。此时，在浏览器输入 http://localhost:4000 回车即可浏览网页效果

安装主题

  • hexo网站模板： https://hexo.io/themes/
  • hexo中较为常用的模板为“NEXT”，我使用的模板为 https://siricee.github.io/hexo-theme-Chic/
  • 一般会到该主题的github进行下载，并将主题文件夹放在myBlog/themes中，在_config.yml文件中修改为“hexo-theme-Chic”（与该主题文件名一致）
  • 修改后在git bash中执行hexo g命令，再hexo s
  • 再次在浏览器输入 http://localhost:4000 回车即可浏览网页更改后的效果

修改主题配置

到perBlog/myBlog/themes/hexo-theme-Chic/文件夹中，打开_config.yml
可自行根据根据自己的需求更改主题下的内容。
更改了该文件后，再次到perblog文件夹下开启git bash here，输入命令行，以便重新生成页面
hexo g
hexo s

重新配置并开启服务后，再次在浏览器输入 http://localhost:4000 回车即可浏览网页更改后的效果

将博客部署到github上

浏览器登录github并且建立新的仓库，且该仓库名必须为 “用户名.github.io”
配置SSH Key，输入下列命令行
git config –global user.name “Your Name”
gif config –global user.email email@example.com
ssh-keygen -t rsa -C ‘email@example.com‘

完成上述三行命令行后，即可生成ssh key，查看自己的ssh
cat ~/.ssh/id_rsa.pub

首次使用还需要确认并添加主机到本机SSH可信列表。若返回Hi xxx！You’ve successfully authenticed, but Github does not provide shell access. 内容，则证实添加成功
ssh -T git@github.com

登录github添加刚刚生成的SSH Key，电机右上角头像->settings->SSH and GPG keys，建立一个新的SSH Key，标题随意取，Key就需要填写刚刚你生成的SSH Key

接下来需要将Github与本地的网页进行链接。首先打开myBlog文件夹下的_config.yml文件夹下，拉倒文末，填写下列配置
deploy:
type: git
repository: https://github.com/Ng123ng/Ng123ng.github.io.git
branch: main

修改后，安装一个部署插件hexo-deployer-git，打开git bash，输入下列命令
npm install hexo-deployer-git –save

最后执行以下两条命令，g为generate缩写，d为deploy缩写
hexo g # 生成页面
hexo d # 部署到Github

首次运行hexo d时会有窗口跳出要求登录你的github账户
现在浏览此输入用户名.github.io就可以访问刚才的网站了。

发布文章

在myblog文件夹下打开git bash，输入下列命令行
‘hexo new “你的文章标题”

上述命令会在你myBlog/source/posts/文件夹下建立一个“你的文章标题.md”的文件，可用notepad打开该文件，该文件支持markdown语法，将你的文章放到该文件中。

修改该文件后，开启git bash，输入下列指令
hexo g //生成页面
hexo s //在本地查看页面是否正确
hexo d //部署到github上
